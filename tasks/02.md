# Template de Prompt para Criar uma Rota de API

Crie a rota `GET /home`

## Descrição

Essa rota deve retornar os dados necessários do usuário autenticado para preencher a página inicial

## Requisitos Técnicos

- Receba uma data no formato YYYY-MM-DD como param da rota
- Utilize a lib dayjs para manipular e formatar datas

## Autenticação

- Rota protegida.
- Apenas o dono (usuário) pode ver as suas informações.

## Response

```ts
interface StatusCode201 {
  activeWorkoutPlanid: string;
  todayWorkoutDay: {
    id: string;
    name: string;
    isRest: boolean;
    weekDay: string;
    estimatedDurationInSeconds: string;
    exercisesCount: number;
  };
  workoutStreak: number;
  consistencyByDay: {
    // key: "2025-01-01" - Incluir todos os dias, mesmo que não possuam sessão
    [key: string]: {
      workoutDayCompleted: boolean;
      workoutDayStarted: boolean;
    };
  };
}
```

## Regras de Negócio

- A partir da data recebida, calcular o início (domingo) e fim (sábado) da semana em UTC.
- Para recuperar o teino do dia, faça o seguinte: 1) Procure pelo workout plan ativo; 2) procure nos dias de treino do workout plan ativo o dia correspondente à data atual (recebida na rota).
- workoutStreak: número de dias em sequência que o usuário completou algum dia de treino (incluído dia de descanso). Exemplo: se o plano de treino tem segunda, terça, quarta e quinta e o usuário completou todos (incluindo descanso) o streak dele é 4

### Montagem do `consistencyByDay`

- Buscar todas as UserWorkoutSession do usuário onde startedAt está dentro do range de domingo 00:00:00 até sábado 23:59:59 da semana correspondente.
- Agrupar as sessões por data (parte da data de startedAt).
- Se uma sessão de treino for completo (completedAt preenchido), ele deve ser contabilizado tanto em workoutDayCompleted quanto em workoutDayStarted.
- Se uma sessão de treino foi apenas iniciado (sem completedAt), ele deve ser contabilizado apenas em workoutDayStarted.
